# 🥖 laCrypta — Sandwich POS System
 
A Java console-based point-of-sale application for **La-Crypta**, a custom sandwich shop.
Customers can fully build their own sandwich orders, add drinks and chips, apply promo codes,
and receive a timestamped receipt file on checkout.
 
Built as Capstone 2 for the Pluralsight Java curriculum, this project applies core
**Object-Oriented Programming** concepts: inheritance, polymorphism, abstract classes,
interfaces, enums, and encapsulation.

## 🚀 How to Run
 
### Prerequisites
- Java 17+
- Maven 3.8+
- A terminal that supports ANSI escape codes (macOS Terminal, Windows Terminal, iTerm2)
  > You can run from the IntelliJ console — it doesn't strip ANSI color codes.
### Steps
 
```bash

git clone https://github.com/Estb1296/laCrypta.git
cd laCrypta
 
# 2. Build with Maven
mvn compile
 
# 3. Run from the target/classes directory
cd target/classes
java com.pluralsight.App
```

## 🏗️ Key Design Decisions
 
### 1. `calculatePrice()` is the single source of truth
 
All price logic lives in `calculatePrice()` — setters never mutate price directly.
`getPrice()` on `MenuItem` always delegates to `calculatePrice()`, so the total
is always recomputed fresh from the current state.
 
```java
// Sandwich.java — price is never stored, always computed
@Override
public double calculatePrice() {
    double total = size.getPrice();           // base price
    if (meat != null && !meat.equalsIgnoreCase("None"))
        total += getMeatPremium();            // size-based meat premium
    if (cheese != null && !cheese.equalsIgnoreCase("None"))
        total += getCheesePremium();          // size-based cheese premium
    for (Topping t: toppings)
        total += t.price();                   // free or premium toppings
    for (ExtraTopping e: extraToppings)
        total += e.getPrice();                // extra meat/cheese upcharge
    return total;
}
```
 
### 2. Extra cheese costs 50% less than extra meat
 
The `isCheese` flag on `ExtraTopping` drives a discount inside `calculatePrice()`:
 
```java
// ExtraTopping.java
@Override
public double calculatePrice() {
    if (size == null) return 0.0;
    if (isCheese) {
        return size.getPrice() * 0.50;  // 50% discount for extra cheese
    }
    return size.getPrice();             // full rate for extra meat
}
```
 
### 3. Signature Sandwiches via inheritance
 
`BLT` and `PhillyCheeseSteak` extend `Sandwich` and set their own defaults in the
constructor, so the customer always starts from a correct template:
 
```java
// BLT.java
public class BLT extends Sandwich {
    public BLT() {
        super("BLT", SandwichSize.EIGHT, "White", true);
        this.setMeat("Bacon");
        this.setCheese("Cheddar");
        this.addSauce("Ranch");
        this.addTopping(new Topping("Lettuce", 0.0, false));
        this.addTopping(new Topping("Tomato", 0.0, false));
    }
}
```
 
### 4. Coupon discount is floored at $0.00
 
The `Order.calculatePrice()` uses `Math.max` to prevent negative totals:
 
```java
// Order.java
@Override
public double calculatePrice() {
    double subtotal = items.stream()
                           .mapToDouble(MenuItem::getPrice)
                           .sum();
    return Math.max(0.0, subtotal - couponDiscount);
}
```
 
### 5. Receipts are timestamped automatically
 
```java
// ReceiptWriter.java
String filename = LocalDateTime.now()
    .format(DateTimeFormatter.ofPattern("yyyyMMdd-HHmmss")) + ".txt";
Path path = Paths.get("receipts", filename);
Files.createDirectories(path.getParent());
```

## 📌 Interesting Code — `toggleTopping()`
 
The most reusable method in the project. Used to let customers add/remove
toppings on Signature Sandwich templates without needing separate add/remove screens:
 
```java
// Sandwich.java
public void toggleTopping(Topping topping) {
    boolean removed = this.toppings.removeIf(
        t -> t.name().equalsIgnoreCase(topping.name())
    );
    if (!removed) {
        this.toppings.add(topping);
    }
}
```
 
If the topping is already on the sandwich, it removes it; if it's missing, it adds it.
One method handles both directions — no branching logic needed in the UI layer.
